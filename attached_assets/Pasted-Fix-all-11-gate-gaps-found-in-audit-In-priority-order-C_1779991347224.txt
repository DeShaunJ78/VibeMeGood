Fix all 11 gate gaps found in audit.
In priority order.

CRITICAL FIX 1 — Prior-only props getting
PLAY/WATCH tags in external-odds.ts

Find this line:
const hardNoPlay = noPlayReason &&
  noPlayReason !== "insufficient_data";

Replace with:
const hardNoPlay = noPlayReason != null;

This means insufficient_data is now a
hard NO-PLAY. Prior-only projections
cannot receive PLAY or WATCH tags ever.

CRITICAL FIX 2 — Optimizer recommending
prior-only props in slate-board.tsx

Find the runOptimizer function where
candidates are filtered.
Add to the filter:
- r.ourProjection?.noPlayReason == null
- r.ourProjection?.gamesUsed != null &&
  r.ourProjection.gamesUsed >= 5

No player with noPlayReason set or
fewer than 5 game logs can appear in
optimizer recommendations.

HIGH FIX 3 — P(Over) big number in
prop-detail-sheet.tsx shows prior-only

When op.sourceLabel === "prior_only"
OR op.gamesUsed === 0 OR op.gamesUsed == null:
Replace the large colored P(Over) number
with a grey "Insufficient data" text.
Do not show a colored percentage.
Show instead: "— Not enough games logged"

HIGH FIX 4 — Hit rate windows with
minimum sample gate in prop-detail-sheet.tsx

For each window (last10, last30, season,
vsOpponent):
Only render the colored bar and percentage
when window.total >= 5.
When total < 5 show grey text:
"Only {total} game(s) — need 5+ for signal"
Do not show the colored bar or percentage.

HIGH FIX 5 — POverBadge in slate-board.tsx
shows prior-only pOver as real signal

When row.ourProjection?.sourceLabel ===
"prior_only" OR gamesUsed < 5:
Do not render POverBadge.
Show a grey dash instead.
Or show a grey "—" with tooltip:
"Insufficient game log data"

MEDIUM FIX 6 — WhyThisEdgePanel variance
scores in prop-detail-sheet.tsx

Add a data depth disclaimer to the panel:

MEDIUM FIX 8 — projectionGap in journal.tsx

When pick.projectionGap is not null but
pick came from a prior-only projection

Add grey "(prior)" label next to the gap.

Add a small grey tooltip on the gap value:

MEDIUM FIX 9 — marketSupportScore with
single book in external-odds.ts

Change minimum book count from 1 to 2
before computing marketSupportScore.

When bookCount < 2:
marketSupportScore = 50 (neutral)
marketEdge = 0
Do not let a single book drive 40% of
the composite action tag.

MEDIUM FIX 10 — p99 ceiling shows when
prior-only in compute.ts

When sourceLabel === "prior_only":
Do not compute or save p99.
Set p99 = null.
A prior-only ceiling is meaningless.

MEDIUM FIX 11 — Calibration LOW SAMPLE
badge is display-only but PLAY tag still
shows in main table.

When calibrationCount < 30 AND
actionTag === "PLAY":
Change actionTag to "WATCH" in the
prop_scores record.
PLAY requires both model confidence AND
calibration validation.
A prop cannot be PLAY when there are
fewer than 30 calibration records for
that sport/stat type.
