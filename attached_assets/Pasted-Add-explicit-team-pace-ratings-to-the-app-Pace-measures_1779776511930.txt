Add explicit team pace ratings to the app.

Pace measures how many possessions a team
uses per 48 minutes. Higher pace = more
stats for everyone in that game.

STEP 1 — Compute pace from existing game logs
Create lib/analytics/pace.ts

Formula:
Possessions = FGA + (0.44 × FTA) - OREB + TOV

Pace = (Possessions / MinutesPlayed) × 48

function computeTeamPace(gameLogs: GameLog[]): number
Takes all game logs for a team and computes
their season average pace rating.

function computeGamePace(
  team1Pace: number, 
  team2Pace: number
): number
Estimates tonight's game pace:
estimatedPace = (team1Pace + team2Pace) / 2

function getPaceLabel(pace: number): string
Returns human readable label:
Above 105:  "Elite pace" (fastest tier)
102-105:    "Fast pace"
98-102:     "Average pace"
95-98:      "Slow pace"
Below 95:   "Very slow pace"

function getPaceAdjustment(pace: number): number
Returns projection multiplier:
Above 105:  +0.06  (6% boost)
102-105:    +0.03  (3% boost)
98-102:      0.00  (no adjustment)
95-98:      -0.03  (-3% reduction)
Below 95:   -0.06  (-6% reduction)

STEP 5 — Wire into projections
When computing player projections apply
the pace adjustment:

adjustedProjection = baseProjection × 
  (1 + getPaceAdjustment(estimatedGamePace))

Store both:
- projectedValue (pace-adjusted)
- baseProjectedValue (unadjusted)
- paceAdjustment (the multiplier applied)
- estimatedGamePace (the pace used)
