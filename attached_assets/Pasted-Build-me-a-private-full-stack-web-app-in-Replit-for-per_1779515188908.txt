Build me a private full-stack web app in Replit for personal PrizePicks analytics.

CONTEXT
This is a private personal decision-support dashboard, not a public consumer app and not a marketing site.
Its purpose is to help me analyze PrizePicks props across all sports using structured data, projection comparisons, injury/news context, line history, external reference markets, journaling, and review metrics.
The app should feel like a serious internal analytics workstation: fast, data-dense, dark mode, table-first, and built for desktop-first use.
I already prefer a stack like React + Express + Drizzle ORM + Tailwind + PostgreSQL, and I want this app built in that direction inside Replit.

IMPORTANT PRODUCT RULES
- This app is for personal use only.
- Do not build a public landing page or generic SaaS template.
- Do not create fake AI summaries that invent stats or injuries.
- Do not hide decision logic in a black-box score.
- Every main score must be decomposable and explainable.
- The app should support all major PrizePicks sports, but the architecture must make it easy to prioritize NBA, NFL, MLB, NHL, and WNBA first.
- PrizePicks payout logic can vary by lineup composition, so do not hardcode one universal payout table as permanent truth. Build the system so payout assumptions can be stored in configuration and overridden by live captured lineup payout data.
- Build the app so external APIs can be swapped via provider adapters.

CORE ENGINES
Build these service modules:

1. PrizePicks Ingestion Service
- ingest current props from adapter
- normalize into pp_lines
- append pp_line_history snapshots
- detect changes and surface alerts
- support standard, demon, goblin line types

2. External Odds Service
- ingest sportsbook reference lines
- store external_lines
- compute no-vig probabilities where possible
- expose support/deviation data to scoring engine

3. Injury / Lineup Service
- ingest injuries and lineup confirmations
- normalize statuses
- expose player/game context for scoring

4. Projection Engine
Start simple, but structure for future upgrades.
Version 1 projection method:
- baseline rolling average
- recent form weighting
- expected minutes/opportunity adjustment
- teammate absence adjustment
- matchup adjustment
- pace/game environment adjustment where relevant
- home/away modifier
Store feature snapshot used for each projection.

5. Scoring Engine
Compute these scores from 0-100:

Edge Score:
- projection gap
- gap percent
- external line support
- no-vig fair value support

Stability Score:
- expected minutes reliability
- confirmed starter status
- teammate injury impact
- historical volatility
- blowout risk

Market Support Score:
- external line deviation
- line movement direction
- line movement magnitude
- whether PP appears stale vs reference markets

Risk Score:
- unresolved injury risk
- late-news risk
- time to lock
- correlation risk in current entry context if applicable

Final Score:
- weighted combination of the above
- final action tag PLAY / WATCH / PASS

IMPORTANT:
- store reasoning as structured JSON
- show exact sub-factors
- do not use opaque AI-generated scores

6. Alert Engine
Trigger alerts only for meaningful changes:
- line movement >= threshold
- injury/news on watched player
- lineup confirmation near lock
- watched prop downgraded from PLAY to WATCH/PASS
- line moves against a submitted or planned entry
- stale data provider
