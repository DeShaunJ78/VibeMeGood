ADDITION 9 — Ensemble Projection Blending
In artifacts/api-server/src/lib/projection/compute.ts

The calibration table is probability_calibration
with columns: sport, stat_type, hit_count,
sample_size, hit_rate.

After computing the weighted mean add
this blending logic:

Query calibration for this sport + statType:
const [calRow] = await db
  .select()
  .from(probabilityCalibrationTable)
  .where(and(
    eq(probabilityCalibrationTable.sport, sport),
    eq(probabilityCalibrationTable.statType, statType),
  ))
  .orderBy(desc(probabilityCalibrationTable.sampleSize))
  .limit(1);

const calCount = Number(calRow?.sampleSize ?? 0);
const calHitRate = calRow?.hitRate
  ? Number(calRow.hitRate) : null;

Blending rules using calCount:
- calCount < 100:
  blendedMean = mean (no change)
  ensembleBlendPct = 0
  sourceLabel unchanged

- calCount 100-299:
  Only blend if calHitRate is meaningful
  calMeanEstimate = ppLine * (calHitRate / 0.55)
  blendedMean = (mean * 0.70) +
                (calMeanEstimate * 0.30)
  ensembleBlendPct = 30
  sourceLabel = "ensemble_30"

- calCount >= 300:
  calMeanEstimate = ppLine * (calHitRate / 0.55)
  blendedMean = (mean * 0.30) +
                (calMeanEstimate * 0.70)
  ensembleBlendPct = 70
  sourceLabel = "ensemble_70"

Use blendedMean instead of mean going forward.

Add to ProjectionOutput interface:
ensembleBlendPct: 0 | 30 | 70;
calSampleSize: number;

Add to our_projections table:

ADDITION 11 — Entry Portfolio Optimizer
The simulation engine already exists in
lib/simulation/entry-simulator.ts with
simulateEntry() function.
Use it directly — do not rebuild it.

STEP 1 — Backend route
Add POST /api/portfolio/optimize to
a new file:
artifacts/api-server/src/routes/portfolio.ts

Request body:
{
  props: Array<{
    playerId: number;
    statType: string;
    lineValue: number;
    lineType: string;
    direction: "more" | "less";
    playerName: string;
    sport: string;
    team: string;
    gameId: string;
    position?: string;
    mean: number;
    stdDev: number;
    modelProb: number;
    vor: number | null;
  }>;
  entrySize: 2 | 3;
  maxEntries: number; // 3-5
}

Portfolio construction logic:
1. Generate all combinations of
   entrySize props from the input props
2. For each combination run simulateEntry()
   with 1000 runs
3. Score each combination:
   portfolioScore = adjustedEV * 100 +
     (vor average of legs * 10) -
     correlationPenalty
   correlationPenalty = 20 if any two legs
     have same team and same direction
4. Sort by portfolioScore descending
5. Select top maxEntries combinations
   ensuring no two selected entries
   share more than 1 player
6. Return selected entries with:
   - legs array
   - entryType (power/flex)
   - multiplier
   - trueJointProbability
   - adjustedEV
   - portfolioScore

Response:
{
  entries: PortfolioEntry[];
  totalPortfolioEV: number;
  generated: number; // how many combos evaluated
}

Register route in index.ts:
import portfolioRouter from "./routes/portfolio";
app.use("/api", portfolioRouter);

## Portfolio Optimizer
The Build Portfolio button in Entry Builder
generates multiple optimized entries from
your selected props simultaneously.

How it works:
1. Add 3-8 props to Entry Builder
2. Click Build Portfolio
3. Select entry size (2 or 3 picks)
4. Select max entries (3-5)
5. System evaluates all combinations
   using the Monte Carlo simulation engine
6. Returns highest EV combinations
   that do not overlap heavily

Portfolio rules:
No two entries share more than 1 player.
Positive correlations within entries boosted.
Payout shift triggers penalized.
All entries must have positive EV.

Unlocks after 30 paper trades logged.
