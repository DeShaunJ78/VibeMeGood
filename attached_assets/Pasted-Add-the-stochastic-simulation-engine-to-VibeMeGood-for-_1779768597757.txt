Add the stochastic simulation engine to VibeMeGood
for true joint probability on multi-leg entries.

CRITICAL RULES:
- This is a TOGGLE — both numbers show side by side
- Independent probability always shows (current math)
- Simulated probability shows alongside it
- Neither replaces the other
- Switch is OFF by default

STEP 1 — Distribution engine
Create artifacts/api-server/src/lib/simulation/distributions.ts

function sampleNormal(mean: number, stdDev: number): number
function sampleLognormal(mean: number, stdDev: number): number
function samplePoisson(lambda: number): number
function sampleNegativeBinomial(mean: number, stdDev: number): number

Always enforce floor = 0. Never return negative values.

Distribution map by sport and stat:
NBA: points=normal, rebounds=normal, assists=normal,
     threes=poisson, steals=poisson, blocks=poisson
NFL: passingYards=lognormal, rushingYards=lognormal,
     receivingYards=lognormal, touchdowns=poisson,
     receptions=normal
MLB: hits=negative_binomial, homeRuns=poisson,
     rbis=negative_binomial, strikeouts=negative_binomial
NHL: goals=poisson, assists=poisson,
     shotsOnGoal=negative_binomial

computeStdDev(gameLogs: number[]): number
If fewer than 5 logs return mean * 0.35 as fallback.

STEP 3 — Entry simulation engine
Create artifacts/api-server/src/lib/simulation/entry-simulator.ts

interface EntryLeg {
  playerName: string
  statType: string
  line: number
  side: "over" | "under"
  modelProb: number
  sport: string
  team: string
  gameId: string
}

async function simulateEntryProbability(config: {
  legs: EntryLeg[]
  runs: number
  multiplier: number
}): Promise<{
  trueJointProbability: number
  naiveProbability: number
  correlationAdjustment: number
  entryEV: number
  adjustedEV: number
  runCount: number
}>

Implementation:
1. For each leg build distribution:
   - mean: from ourProjectionsTable
   - stdDev: from player_game_logs
   - distribution: from DISTRIBUTION_MAP
   - floor: 0

2. Build correlation matrix from leg context
   (same team, same game, sport, position)

3. Run simulation (default 10,000 runs):
   Each run samples every leg from distribution
   accounting for correlations.
   Count how many runs ALL legs hit simultaneously.

4. trueJointProbability = allHit / runs
5. naiveProbability = product of all modelProbs
6. correlationAdjustment = true - naive
7. entryEV = trueJointProbability * multiplier - 1
8. adjustedEV = entryEV (with payout shift applied
   if detectPayoutShift() flags the entry)
