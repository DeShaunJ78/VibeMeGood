Add a Shark Chat page to VibeMeGood.
Same pattern as DraftDuel's /shark page.
This is a completely separate chat from DraftDuel.

STEP 1 — Create SharkChatContext
Create artifacts/prizepicks/src/contexts/SharkChatContext.tsx

Use localStorage for persistence.
Storage keys must be DIFFERENT from DraftDuel:
  "vmg_shark_messages" (not "shark_messages")
  "vmg_shark_draft_input" (not "shark_draft_input")

This prevents DraftDuel and VibeMeGood chat 
histories from mixing.

Interface:
- messages: ChatMessage[]
- draftInput: string
- setDraftInput: (val: string) => void
- sendMessage: (text: string) => void
- clearChat: () => void
- isLoading: boolean

Welcome message:
"Welcome. I'm your Personal Shark for VibeMeGood.
Ask me anything about PrizePicks pick'em analytics,
entry construction, break-even rates, payout shifts,
variance signals, or the daily workflow.
Paper Trade Mode is active — no real-money 
recommendations."

Wrap the provider at the top level in App.tsx
so it never unmounts during navigation.

STEP 2 — Create the Shark Chat page
Create artifacts/prizepicks/src/pages/shark-chat.tsx

Same layout as DraftDuel:
- Dark theme matching VibeMeGood design
- "🦈 Personal Shark" header
- Paper Trade Mode banner (permanent, top of chat)
- User messages on right, assistant on left
- Typing indicator while waiting for response
- Auto-scroll to bottom on new messages
- Input field at bottom with send button
- Shift+Enter for new line, Enter to send
- Trash icon in top right → inline confirmation
  "Clear this conversation? [Clear] [Cancel]"
  Clears localStorage on confirm

STEP 3 — Backend route
The existing /api/shark/chat endpoint may already
exist. Check first.

If it exists, verify it accepts an "app" parameter
that controls which knowledge folders are searched.

  "conversationHistory": [

}

Response:
{ "reply": "string" }
