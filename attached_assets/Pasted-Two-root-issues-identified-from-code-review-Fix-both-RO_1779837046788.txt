ROOT ISSUE 1 — Projection gap showing 20.0
The market-intel route joins ourProjectionsTable
by playerId + statType. This only matches
for basic stat types like "Points", "Rebounds",
"Assists".

But PP lines have exotic stat types like:
"Points - 1st 3 Minutes"
"Free Throws Made"
"Free Throws Attempted"
"Rebs+Asts"
"Pts+Asts"
"Blocked Shots"
"Two Pointers Made"

The projection scraper only saves these
basic stat types per player:
Points, Rebounds, Assists, Steals, Blocks,
Turnovers, Three-Pointers Made

So the join fails for every exotic stat type
and 20.0 shows as the placeholder gap.

FIX: In the market-intel route, when joining
ourProjectionsTable, also try to match on
the BASE stat type when the full stat type
does not match.

Create a function that extracts the base
stat from exotic PP line stat types:

function getBaseStatType(statType: string): string {
  if (statType.includes("Points")) return "Points"
  if (statType.includes("Rebound")) return "Rebounds"
  if (statType.includes("Assist")) return "Assists"
  if (statType.includes("Steal")) return "Steals"
  if (statType.includes("Block")) return "Blocks"
  if (statType.includes("Three") || statType.includes("3-PT")) return "3-Pointers Made"
  if (statType.includes("Free Throw")) return "Free Throws Made"
  if (statType.includes("Turnover")) return "Turnovers"
  return statType
}

Then in the market-intel result loop when
proj is null (no projection found), try
fetching projection by playerId + base stat type.

Also the "Rebs+Asts" combo lines should
use Rebounds projection + Assists projection
added together as the projected value.

Same for "Pts+Asts", "Pts+Rebs",
"Pts+Rebs+Asts" combo lines.

Add a function:
function getComboProjection(
  statType: string,
  projections: Map<string, number>
): number | null {
  Map keys are base stat types.
  "Rebs+Asts" = projections.get("Rebounds") 
                + projections.get("Assists")
  "Pts+Asts" = projections.get("Points")
               + projections.get("Assists")
  "Pts+Rebs" = projections.get("Points")
               + projections.get("Rebounds")
  "Pts+Rebs+Asts" = Points + Rebounds + Assists
  "Hits+Runs+RBIs" = Hits + Runs + RBIs
  Return null if any component is missing.
}

Build a projections map per player before
the result loop so combo lookups are fast.
